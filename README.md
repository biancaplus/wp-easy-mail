# WP Easy Mail

一个轻量、独立的 WordPress 多表单插件，用于创建「联系我们」和「获取报价」表单，并管理邮件通知与提交记录。

## 基本信息

- 作者：`biancaplus`
- 当前版本：`1.1.0`
- WordPress：`6.0+`
- PHP：`7.4+`
- 许可证：[GPL-2.0-or-later](https://www.gnu.org/licenses/gpl-2.0.html)
- 页面构建器：无依赖

## 功能

- 创建多个独立表单，并通过短代码插入文章或页面。
- 提供「联系我们」和「获取报价」两种表单类型。
- 支持多套前台外观模板（经典 / 卡片），后台可选并实时预览。
- 可配置主题色，同步用于提交按钮、合规链接与 HTML 通知邮件。
- 每个表单可独立配置描述、按钮文案、邮件、提示消息和字段。
- 电话和公司字段可分别设置为显示/隐藏、必填或选填。
- 提交记录页采用简洁列表：未读高亮、信息弹窗、一键已读/删除。
- 记录「表单类型」标签与「操作人」（已读等操作对应的登录用户）。
- HTML 邮件采用卡片式布局，提交详情按表单可见字段自动生成。
- 支持 reCAPTCHA v3、蜜罐和提交速率限制；启用时提供合规说明文案并隐藏官方徽章。
- 支持 WordPress 个人数据导出与擦除工具。
- 使用 WordPress 原生 `wp_mail()` 接口发送邮件。

## 安装

1. 将 `wp-easy-mail` 目录上传至 `/wp-content/plugins/`。
2. 在 WordPress 后台的「插件」页面启用 **WP Easy Mail**。
3. 前往「联系我们 > 表单管理」创建并发布表单。
4. 复制表单短代码并添加到文章或页面，例如：

```text
[wp_easy_mail id="123"]
```

## 表单外观模板

在表单编辑页的「表单设置」中可选择模板，并查看对应预览：

| 模板 | 说明 |
|------|------|
| 经典（`classic`） | 简洁竖排布局，适合多数页面嵌入。 |
| 卡片（`card`） | 圆角白卡片 + 姓名/电话双列，适合独立联系页展示。 |

主题色会同步到预览与前台按钮、卡片模板焦点环及 reCAPTCHA 合规链接。

## 表单类型

### 联系我们

- 姓名
- 电子邮件
- 电话
- 公司（可选显示）
- 信息

### 获取报价

- 姓名
- 电子邮件
- 电话
- 公司（可选显示）
- 需求描述
- 预算

两种表单中的电话和公司字段均可通过后台开关设置为必填或选填；「联系我们」默认隐藏公司字段。

## 邮件投递配置

WP Easy Mail 不内置 SMTP 服务，邮件通过 WordPress 原生 `wp_mail()` 接口发送。

为了确保邮件可以稳定投递，需要安装并正确配置 **WP Mail SMTP**：

1. 在 WordPress 后台安装并启用 **WP Mail SMTP**。
2. 根据所使用的邮件服务商填写 SMTP 主机、端口、加密方式、账号和密码。
3. 在 WP Mail SMTP 中发送测试邮件，确认投递成功。
4. 返回 WP Easy Mail，为表单配置收件人、主题及发件信息。

未配置可用的邮件发送服务时，表单提交仍会保存到后台，但邮件状态可能显示为「发送失败」。前台成功提示表示提交已入库，不代表邮件一定投递成功。

### HTML 通知邮件

- 使用主题色顶栏的卡片式 HTML 邮件（纯文本格式仍可选择）。
- **提交详情按当前表单类型的可见字段自动组装**（含需求描述、预算等），不依赖可能过期的自定义正文模板。
- 「邮件正文（备用）」仅作参考预览；切换表单类型时，若仍是默认模板会自动同步文案。

## 提交记录

路径：`联系我们 > 提交记录`

- 显示 ID、表单类型标签、姓名、电话、邮箱、信息摘要、提交时间、已读状态、邮件状态、操作人与操作。
- 点击信息摘要可弹出完整内容；未读记录会自动标为已读并记录操作人。
- 不在后台列表中展示访客 IP（IP 仅用于防滥用限流等内部用途）。

## reCAPTCHA v3

前往「联系我们 > reCAPTCHA v3」完成以下配置：

1. 填写 Site Key。
2. 填写 Secret Key。
3. 设置验证分数阈值。
4. 启用 reCAPTCHA v3。

启用时必须同时提供匹配的 Site Key 和 Secret Key。启用后前台显示合规说明文案，并按 Google 建议隐藏 `.grecaptcha-badge` 徽章。

## 常见问题

### 是否依赖 Elementor 或其他页面构建器？

不依赖。插件使用标准 WordPress 短代码。

### 是否内置 SMTP？

不内置。请安装并配置 **WP Mail SMTP**，底层邮件仍通过 `wp_mail()` 发送。

### 邮件发送失败后，提交记录会丢失吗？

不会。提交记录会先保存到数据库，后台会单独记录邮件发送状态。

### 为什么页面显示提交成功，但后台写「发送失败」？

提交入库与邮件投递是分开的。请检查 WP Mail SMTP 配置（主机、端口与加密方式需匹配，例如 465 配 SSL、587 配 TLS），并在 SMTP 插件中完成测试邮件。

### 如何更换表单外观？

编辑对应表单 →「表单设置」→ 选择「经典」或「卡片」模板并保存。
